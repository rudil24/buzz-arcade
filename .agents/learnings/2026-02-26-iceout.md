# Learnings: Ice Out (WebAssembly)

**Date:** 2026-02-26

## Learning 1: Pygbag + Vite Bootloader Integration
**ID**: L1
**Category**: deployment
**Confidence**: high
**Observation:** Newer versions of Pygbag bundle `tarfile` and `posix` path extractions in their generated `index.html` files which fail entirely in Vite/React local environments, throwing `tarfile.ReadError` or `AttributeError`.
**Standard Pattern:** Edit Pygbag's `index.html` bootloader template physically. Force it to run `import zipfile` logic across all domains (bypassing its `localhost` checks) and use `str(main_path)` when invoking `shell.source()`. Package assets into `.apk` zipfiles natively instead.

## Learning 2: Pygame Exit vs Iframe Architecture
**ID**: L2
**Category**: architecture
**Confidence**: high
**Observation:** `sys.exit()` or `quit()` simply crash the WebAssembly container, appearing to the user as a frozen, unresponsive page.
**Standard Pattern:** Detect the Pygbag `platform.window` and trigger Javascript `postMessage('exit_signal')` upward to the parent DOM wrapper. Let the React/Vue layer handle the unmounting or navigation.

## Learning 3: WebAssembly GitHub Pages Threading
**ID**: L3
**Category**: deployment
**Confidence**: high
**Observation:** `SharedArrayBuffer` is mandatory for Pygame to render audio and complex loop events, but requires `Cross-Origin-Embedder-Policy: require-corp` headers to activate in Chromium browsers. GitHub Pages forbids editing HTTPS headers.
**Standard Pattern:** Emulate a reverse proxy using a service worker intercept. Implement the `coi-serviceworker` (gzuidhof/coi-serviceworker) in the `<head>` of the `index.html` output to artificially synthesize the isolation rules required.

## Learning 4: NanoBanana Art Pipeline inside Google Antigravity
**ID**: L4
**Category**: architecture
**Confidence**: high
**Observation:** Relying on basic Python PIL filters to generate 8-bit sprites resulted in subpar, clunky art despite good initial mocks.
**Standard Pattern:** The OPST stack specifically running inside the Google Antigravity ecosystem must prioritize invoking the **NanoBanana API** via agent-directed requests directly instead of forcing the Product Owner to generate assets manually in a separate Chrome tab.

## Learning 5: Mobile Compatibility for Web Emulators
**ID**: L5
**Category**: architecture
**Confidence**: high
**Observation:** The game physics and gameplay were excellent on desktop, but deploying a web-wrapper game with only keyboard event listeners makes it completely unplayable on iOS/Android devices.
**Standard Pattern:** All future web-based game UI/UX deliverables must explicitly include an on-screen D-Pad or touch-gesture event-listener mapping (mobile controls) layered on top of the WASM canvas.

## Learning 6: The Necessity of Web Audio
**ID**: L6
**Category**: architecture
**Confidence**: medium
**Observation:** A silent arcade game fails to capture the intended retro aesthetic, actively deterring player engagement even if they play "on mute" later.
**Standard Pattern:** Sound design and audio integration must be treated as a blocking `P0` requirement for MVP shipment in all future game projects.
