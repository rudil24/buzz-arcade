# Retro: Ice Out (WebAssembly Arcade)
**Date:** 2026-02-26
**Lead:** Cap

## Objective
The goal was to stabilize, test, and deploy a Python/Pygame application ("Ice Out") into a modern React (Vite) styled web wrapper, and host it on GitHub Pages. 

## Successes
1. **React Integration:** Embedding a Pygame canvas within a Vite iframe works perfectly as a mechanism for deploying retro-styled arcades.
2. **Behavioral Pytest:** Extracting state transitions out of the primary `while True` game loop allowed us to inject mocked keystrokes into small pure functions, establishing an excellent 18% Core Logic test coverage metric.
3. **Algorithmic Art:** We utilized simple PIL (Python Imaging Library) tools alongside `rembg` AI background removal to dynamically generate completely transparent, true 16-color 8-bit sprites without manual Photoshop work.

## Hurdles & Fails
1. **Pygbag `sys.exit()`:** Using standard Python exit commands in a WASM Pygame loop entirely freezes the browser execution thread without warning the parent DOM.
2. **Viewport Scaling on Initial Mount:** Because Vite's DOM renders almost instantly, Pygbag's `window.innerWidth` detectors miscalculated canvas sizes before CSS layouts locked them in.
3. **CORS/COEP Security Headers:** Browsers enforce strict Cross-Origin Isolation when launching WebWorker threads, which GitHub Pages explicitly strips. This prevented the Pygbag engine from ever starting.

## Resolutions
- **Scaling Fix:** Deeply injected `dispatchEvent(new Event('resize'))` from the `App.jsx` React lifecycle hook into the child Pygame iframe to force hardware scaling sync.
- **Escape Loop Fix:** We created an environment bridge. When the player pauses and double-escapes, the Python script executes `platform.window.parent.postMessage()`, allowing the React frontend to cleanly unmount the emulator.
- **Security Headers Fix:** We hosted a `coi-serviceworker.js` file locally to actively rewrite network headers over the Vite dev server and on the GitHub host.
