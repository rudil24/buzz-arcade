# Retro: Ice Out (WebAssembly Arcade)

**Date:** 2026-02-26
**Lead:** Cap

## Objective

The goal was to stabilize, test, and deploy a Python/Pygame application ("Ice Out") into a modern React (Vite) styled web wrapper, and host it on GitHub Pages.

## What Went Well

- **React Integration & Deployment:** Embedding a Pygame canvas within a Vite iframe worked excellently. Despite a few hurdles, the OPST successfully stabilized the pipeline and pushed the React wrapper and game natively to GitHub Pages.
- **Physics Engine & Gameplay:** The core Python gameplay logic and physics handling were solid and playable out-of-the-gate.
- **Behavioral Pytest:** Extracting state transitions out of the primary `while True` game loop allowed us to inject mocked keystrokes into small pure functions, establishing an excellent 18% Core Logic test coverage metric.
- **Cross-Origin Bridge:** Discovered an elegant architecture for allowing the interior Python WASM loop to trigger environment unmounts by passing `postMessage` calls up to the host React DOM.

## What Could Be Improved

- **8-Bit Art Generation Pipeline:** The OPST failed to properly access and utilize NanoBanana for high-quality pixel art generation. Delivering clunky, low-quality script-filtered sprites after promising high-fidelity 8-bit aesthetic mocks is unacceptable. The Product Owner had to manually step in to fix the art assets.
- **Mobile Unplayability:** Deploying a web-wrapper game with keyboard-only controls immediately alienated the mobile/tablet market. Touch/on-screen controls must be considered standard for Web deployments.
- **Lack of Sound Design:** A silent arcade game fails the aesthetic vibe-check. Sound is a critical component of the retro arcade experience.
- **Pygbag `sys.exit()` and Checkpoints:** WebAssembly Python crashes without warning when invoking standard exit routines, and initial viewport measurements miscalculate due to React's rapid asynchronous CSS DOM mounting.
